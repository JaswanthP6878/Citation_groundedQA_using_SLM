%% ── Preamble ─────────────────────────────────────────────────────────────────
\noindent\textbf{Project type:}
Empirical NLP project focused on retrieval-augmented generation (RAG) for
hallucination reduction.

\noindent\textbf{Research question:}
\emph{Within a fully local, privacy-preserving RAG system, how much of a
sub-7B SLM's hallucination rate is attributable to retrieval quality versus
generation capacity?}

%% ── 1. Problem Statement ─────────────────────────────────────────────────────
\section{Problem Statement \& Motivation}

Hallucination generating claims unsupported by source documents is the
primary failure mode of LLMs in high-stakes settings.
Cloud tools like Google's NotebookLM require transmitting corpora to external
servers, conflicting with HIPAA/GDPR requirements.
\textbf{Cloud upload equals data exfiltration risk.}
Sub-7B SLMs (e.g., Phi-3-mini-3.8B \citep{microsoft2024phi3}) can run on a
single consumer GPU, making local deployment feasible.

\textbf{Task definition.}
Given a corpus $\mathcal{C}=\{d_1,\ldots,d_n\}$ and query $q$, the system
produces answer $a$ plus, for every sentence $s_i\in a$, a citation
$(p_i, s_i)$ where passage $p_i\in\mathcal{C}$ textually entails $s_i$.
Any sentence without a valid citation is counted as a hallucination.

%% ── 2. Proposed System ───────────────────────────────────────────────────────
\section{Proposed System}

All components run locally; no external API calls are required.

\textbf{Parsing.}  QASPER provides structured JSON with section boundaries,
which we load directly.

\textbf{Chunking (ablated).}  Three strategies: fixed-size (256 tok, 32 overlap),
sliding window (512 tok, 128 overlap), and section-aware
(boundary-aligned, capped at 512 tokens).

\textbf{Retrieval (ablated).}  Dense only (FAISS + BGE-small-en-v1.5);
BM25 (rank-bm25); hybrid reciprocal-rank fusion (RRF); hybrid +
cross-encoder reranking (ms-marco-MiniLM-L-6-v2), top-5 passages.

\textbf{Generation.}  Two prompt variants: standard generation vs.\
\emph{citation forcing}, where the model must bracket every claim with an
inline citation and assert only what the retrieved passages support.
Served via Ollama; $\leq$16 GB VRAM.

\subsection{Dataset \& Baselines}

\textbf{Dataset:} QASPER \citep{dasigi2021qasper} 1,585 NLP papers,
5,049 Q\&A pairs with gold evidence annotations supporting both
retrieval and answer evaluation (official token F1).

\textbf{Baselines:} (1)~Parametric-only Phi-3-mini; (2)~BM25-only; (3)~Random retrieval (5 passages).

\subsection{Metrics \& Hypothesis}

\textbf{Primary:} Citation precision via NLI entailment (TRUE cross encoder) and hallucination rate.
\textbf{Secondary:} Token F1 (QASPER official), BERTScore.
\textbf{Faithfulness:} RAGAS \citep{es2023ragas} (Llama-3-8B local judge).
\textbf{Retrieval:} Recall@5, MRR@10, evidence hit rate.

\textbf{Hypothesis:} Retrieval improvements (A$\to$D) will account for the
majority of hallucination reduction, with citation forcing (D$\to$E) providing
a smaller incremental gain; results will be stratified by whether gold evidence
was retrieved to attribute failures to retrieval vs.\ generation.
